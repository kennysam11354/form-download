# Pest Control ERP v7: 사용자 매뉴얼 (User Manual)

**정확한 데이터 관리와 신속한 청구를 위한 단계별 가이드**

---

## 1. 인보이스(Invoice) 작성 및 발행

**현장에서 완료된 작업 기록(Ticket)을 묶어 고객에게 공식 청구서를 발행합니다.**

### 📋 작성 단계
1.  **대상 선택:** 관리자 대시보드에서 `Invoices` 메뉴로 접속한 후 `Create New Invoice`를 클릭합니다.
2.  **고객사 지정:** 청구할 고객사(Account)를 선택합니다. 시스템이 해당 고객사의 '미납 작업(Submitted Tickets)' 목록을 자동으로 불러옵니다.
3.  **티켓 검토 및 포함:** 
    *   목록에서 이번 청구에 포함할 작업 티켓을 체크합니다.
    *   특정 작업을 이번 청구에서 제외하려면 `Exclude` 버튼을 누르고 사유를 입력합니다. (제외된 기록은 향후 추적 가능)
4.  **세부 사항 확인:**
    *   **Payment Term:** 고객사 설정에 따라 납부 기한(Due Date)이 자동 계산됩니다. (필요 시 수동 수정 가능)
    *   **Tax:** 해당 지역 및 고객사 세율이 자동으로 적용됩니다.
5.  **발행(Issue):** `Issue Invoice` 버튼을 클릭합니다. 
    *   **⚠️ 주의:** 발행된 인보이스는 회계 원장(Ledger)에 즉시 기록되며, 이후에는 **수정이 불가능**합니다. (수정이 필요한 경우 Void 후 재출력해야 함)

## 2. 인보이스 작성 시 현장 사진 첨부 및 관리

**작업의 투명성을 높이고 고객의 신뢰를 얻기 위해 현장 사진을 청구서에 포함합니다.**

### 📸 사진 관리 방법
1.  **현장 기사 업로드:** 현장 기사가 모바일 앱에서 작업을 마칠 때 `Before/After` 사진을 촬영하여 티켓에 첨부합니다.
2.  **관리자 검토:** 인보이스 작성 단계에서 개별 티켓의 상세 보기(`View Ticket`)를 클릭하여 첨부된 사진을 확인합니다.
3.  **청구서 포함 설정:** 
    *   인보이스 템플릿 설정에서 `Include Service Photos` 옵션을 체크합니다.
    *   이 옵션이 활성화되면, 고객에게 발송되는 인보이스 하단에 해당 작업 기간의 주요 현장 사진이 증빙 자료로 자동 첨부됩니다.
4.  **클라우드 저장:** 모든 사진은 서버(S3)에 안전하게 저장되며, 언제든 과거 인보이스나 티켓을 통해 다시 조회할 수 있습니다.

## 3. 고객사별 미수금(AR) 조회 및 관리

**회사의 재무 건전성을 유지하기 위해 실시간 미수금 현황을 한눈에 파악합니다.**

### 💰 미수금 확인 방법
1.  **종합 대시보드:** 메인 화면의 `AR Aging Summary`에서 30일/60일/90일 이상 연체된 총 미수금을 확인합니다.
2.  **고객사별 상세 조회:** `Accounts` 메뉴에서 특정 고객사를 검색하여 클릭합니다.
    *   **Current Balance:** 해당 고객사가 현재까지 납부하지 않은 총 금액이 실시간으로 표시됩니다.
    *   **Ledger 합산:** 이 금액은 별도의 수치가 아니라, 모든 인보이스(+)와 입금 기록(-)을 합산한 실제 데이터입니다.
3.  **입금(Payment) 등록:**
    *   고객으로부터 대금을 수령하면 `Post Payment`를 실행합니다.
    *   입금액을 입력하면 고객사의 총 미수금(AR)이 즉시 차감됩니다.
4.  **배정(Allocation):** 
    *   입금된 금액을 특정 인보이스에 배정하여 어떤 인보이스가 완납(`PAID`) 되었는지 관리합니다.
    *   배정하지 않은 금액은 `Unapplied Payment`로 남으며 나중에 다른 청구서에 적용할 수 있습니다.

## 4. 서브 컨트랙터 및 고객 구좌 관리 (Account Management)

**오너는 유연하게 고객 구좌를 서브 컨트랙터에게 배분하고 이동할 수 있습니다.**

### 🔄 구좌 배정 및 이동 방법
1.  **신규 구좌 배분:** 새로 생성된 고객(Account)을 대시보드에서 `Assign to Sub-contractor` 옵션을 통해 담당 테크니션에게 배정합니다.
2.  **구좌 간 이동(Transfer):** 기존 고객을 다른 테크니션에게 이관해야 할 경우, `Account Transfer` 메뉴를 사용하여 히스토리를 유지한 채 담당자를 변경합니다. 
3.  **독립적 관리 한계:** 테크니션은 자신에게 배정된 고객 구좌만 조회 및 관리할 수 있으며, 타 서브 컨트랙터의 고객 데이터에는 접근이 불가능합니다.

---

## 5. 지능형 모니터링 및 그룹별 스테이트먼트 (Intelligent Monitoring)

**비즈니스 성과를 다각도로 분석하고 스테이트먼트를 효율적으로 관리합니다.**

### 📊 대시보드 활용 및 필터링
1.  **그룹별 모니터링:** 전체 현황을 **서브 컨트랙터 그룹군별**로 필터링하여 각 그룹의 수익성과 활동량을 비교 분석합니다.
2.  **상세 분류 조회:** 미수금 현황을 **연체 기간별(Aging)**, **매출 규모별(Revenue)** 순으로 정렬하여 우선순위가 높은 관리 대상을 선정한 뒤 즉각적인 조치(연락, 법적 절차 등)를 취할 수 있습니다.
3.  **스테이트먼트 자동 발송:** 
    *   시스템이 발행된 티켓을 바탕으로 스테이트먼트를 자동 산출합니다.
    *   오너는 이를 서브 컨트랙터별로 그룹화하여 검토하거나, 설정된 일자에 맞춰 고객들에게 자동으로 대량 발송합니다.

---

## 💡 유용한 팁 (Pro Tips)
*   **자연어 명령 활용:** 채팅창에 "A 고객사 미수금 리스트 보여줘"라고 입력하면 AI가 즉시 해당 표를 생성해 줍니다.
*   **실시간 업데이트:** 현장 기사가 서명을 받는 즉시 사무실 관리자의 인보이스 대상 목록에 해당 건이 나타납니다.
*   **데이터 무결성:** 실수로 인보이스를 잘못 발행했다면 당황하지 말고 `Void` 버튼을 누르세요. 시스템이 자동으로 마이너스(-) 기록을 생성하여 장부를 바로잡습니다.
